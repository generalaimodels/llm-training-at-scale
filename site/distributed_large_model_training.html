When Mtotal>MGPUM_{\text{total}} > M_{\text{GPU}}Mtotal​>MGPU​ and additional GPUs are unavailable, two primary strategies reduce memory:

Full Activation Recomputation: Instead of storing all intermediate activations during the forward pass, discard them and recompute during the backward pass. This eliminates MactM_{\text{act}}Mact​ at the cost of one additional forward pass:

Compute overhead=Ffwd+Ffwd+FbwdFfwd+Fbwd=2Ffwd+FbwdFfwd+Fbwd\text{Compute overhead} = \frac{F_{\text{fwd}} + F_{\text{fwd}} + F_{\text{bwd}}}{F_{\text{fwd}} + F_{\text{bwd}}} = \frac{2F_{\text{fwd}} + F_{\text{bwd}}}{F_{\text{fwd}} + F_{\text{bwd}}}Compute overhead=Ffwd​+Fbwd​Ffwd​+Ffwd​+Fbwd​​=Ffwd​+Fbwd​2Ffwd​+Fbwd​​

Since Fbwd≈2FfwdF_{\text{bwd}} \approx 2 F_{\text{fwd}}Fbwd​≈2Ffwd​:

6.2 CUDA Memory Allocator Behavior

The CUDA caching allocator (used by PyTorch) maintains pools of allocated memory blocks. Fragmentation occurs when freed blocks cannot be coalesced:

Mallocated>Mused+MfragmentedM_{\text{allocated}} > M_{\text{used}} + M_{\text{fragmented}}Mallocated​>Mused​+Mfragmented​

This means the theoretical memory computation MtotalM_{\text{total}}Mtotal​ underestimates actual GPU memory consumption. In practice:

Mactual≈Mtotal×(1+ϵfrag)M_{\text{actual}} \approx M_{\text{total}} \times (1 + \epsilon_{\text{frag}})Mactual​≈Mtotal​×(1+ϵfrag​)

where ϵfrag∈[0.05,0.20]\epsilon_{\text{frag}} \in [0.05, 0.20]ϵfrag​∈[0.05,0.20] depending on allocation patterns.

NCCL (NVIDIA Collective Communications Library) implements collectives using GPU Streaming Multiprocessors (SMs). This creates a critical hidden cost:

True MFU=Fmodel⋅BglobalTstep⋅NGPU⋅Φpeak<Expected MFU (assuming perfect overlap)\text{True MFU} = \frac{F_{\text{model}} \cdot B_{\text{global}}}{T_{\text{step}} \cdot N_{\text{GPU}} \cdot \Phi_{\text{peak}}} < \text{Expected MFU (assuming perfect overlap)}True MFU=Tstep​⋅NGPU​⋅Φpeak​Fmodel​⋅Bglobal​​<Expected MFU (assuming perfect overlap)
